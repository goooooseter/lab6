# lab6
Описание реализованных классов и интерфейсов
SortingStrategy — абстрактный интерфейс, определяющий метод sort(std::vector<int>&).

BubbleSortStrategy — реализация пузырьковой сортировки.

QuickSortStrategy — реализация быстрой сортировки с использованием рекурсии.

StdSortStrategy — стратегия, использующая встроенную сортировку std::sort из STL.

InsertionSortStrategy — реализация сортировки вставками.

Sorter — класс, принимающий объект SortingStrategy и вызывающий у него метод сортировки. Не зависит от конкретной реализации стратегии.

Объяснение паттерна "Стратегия" и его преимуществ
Паттерн "Стратегия" — это поведенческий шаблон проектирования, использующийся, если требуется применять разные алгоритмы из семейства в разных ситуациях. Их нужно привести к единому интерфейсу для удобства исполнения.

Преимущества:
Гибкость: можно динамически выбирать стратегию в зависимости от условий.

Расширяемость: добавление новых алгоритмов не требует изменения кода уже существующих компонентов.

Чистота кода: логика сортировки изолирована от логики управления.

Соответствие принципам SOLID, особенно OCP и SRP.

SRP (Single Responsibility Principle) - Каждый класс имеет только одну зону ответственности. Стратегии отвечают только за сортировку, Sorter — только за использование стратегии.
OCP (Open/Closed Principle) - Sorter открыт для расширения — можно подключать новые стратегии — и закрыт для модификации — сам Sorter не нужно менять при добавлении новых стратегий.

Выводы о гибкости и расширяемости разработанной системы
Система позволяет добавлять новые алгоритмы сортировки без изменения существующего кода, просто реализовав новый класс, наследующий SortingStrategy.

Благодаря применению паттерна "Стратегия", сортировки легко тестировать и подменять.

Такой подход упрощает поддержку и развитие кода в будущем, делая архитектуру модульной и удобной для командной разработки.
